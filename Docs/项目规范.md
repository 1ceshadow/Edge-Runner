## 项目结构

结构不是固定不变的，怎么方便怎么来，可以随着项目扩展，重新组织。  
一般有两种结构，按相关性分类 和 按功能性分类

我们项目人少，然后有很多重复利用，所以就整体按照功能性(模块化)分类，细节处按照相关性分类    

下面是一个试用结构：
```
ProjectRoot/                                  (项目根目录)
├── Assets/                                   (核心资产文件夹，所有 Unity 导入的资源在此)
│   
|
├── Docs/  (项目文档)
│   ├── StyleGuide.md  (代码和命名规范)
│   └── Architecture.md  (架构图和设计文档)
├── .gitignore  (版本控制忽略文件)
|
|
|   // 下面通常是自动生成的
├── Packages/  (Unity Package Manager 管理的包，通常自动生成)
├── ProjectSettings/  (项目设置文件，如输入、标签等)
├── UserSettings/
├── Library/  (Unity 缓存文件夹，忽略版本控制)
├── Temp/  (临时文件，忽略版本控制)
├── Logs/  (日志文件，忽略版本控制)
└── ProjectName.slnx
```

Assets 结构示例：

**结构一**
```
Assets/
├── Core/                   (核心系统和框架，包含共享脚本和配置)
│   ├── Scripts/            (核心脚本，如游戏管理器、事件系统)
│   ├── Config/             (ScriptableObjects 或 JSON 配置文件)
│   └── Resources/          (动态加载的共享资源，如全局数据)
├── Modules/                (功能模块，按游戏系统分组，便于大型项目模块化)
│   ├── Gameplay/           (游戏核心逻辑)
│   │   ├── Scripts/        (如 PlayerController.cs)
│   │   ├── Prefabs/        (预制体，如 PlayerPrefab)
│   │   ├── Scenes/         (相关场景，如 Level1.unity)
│   │   └── Assets/         (模块特定资源，如模型、纹理)
│   ├── UI/                 (用户界面模块)
│   │   ├── Scripts/        (UI 脚本，如 MenuManager.cs)
│   │   ├── Prefabs/        (UI 预制体)
│   │   └── Assets/         (UI  sprites 和字体)
│   ├── Audio/              (音频模块)
│   │   ├── Scripts/        (音频管理脚本)
│   │   └── Sounds/         (音频文件，如 BGM、SFX)
│   └── Networking/         (如果有多人功能)
│       ├── Scripts/        (网络脚本)
│       └── Prefabs/        (网络相关预制体)
├── Art/                    (美术资源，按类型细分)
│   ├── Models/             (3D 模型)
│   ├── Textures/           (纹理和材质)
│   ├── Sprites/            (2D 精灵)
│   └── Animations/         (动画剪辑)
├── Scenes/                 (所有场景文件，按级别或类型分组)
│   ├── Main/               (主场景，如 Menu.unity)
│   └── Levels/             (关卡场景，如 Level01.unity)
├── Prefabs/                (共享预制体，不属于特定模块的)
|
|
├── Plugins/  (Unity 插件和扩展)
└── 第三方资产包，如从 Asset Store 导入的，他们默认导入到某个位置，就不要再移动了 
```

**结构二**
```
Assets/
├─ Core/                  ← 全局共享、基础系统
│   ├─ Scripts/
│   │   ├─ Manager/       ← GameManager, SaveSystem, EventSystem 等
│   │   ├─ Utilities/     ← 扩展方法、工具类
│   │   └─ Data/          ← ScriptableObject 全局配置
│   └─ Settings/          ← InputSystem 设置、Project Settings 资产等
│
├─ Modules/               ← 按功能模块划分（最大亮点，强烈建议）
│   ├─ Player/
│   │   ├─ Scripts/      ← PlayerController, PlayerHealth 等
│   │   ├─ Prefabs/       ← Player 主预制体及变体
│   │   ├─ Animations/    
│   │   ├─ Audio/         
│   │   └─ Effects/       ← 粒子、特效
│   │
│   ├─ Enemies/           ← 所有敌人共用一个模块
│   │   ├─ Scripts/
│   │   ├─ Prefabs/       ← 各种敌人预制体
│   │   ├─ Animations/
│   │   └─ Data/          ← 敌人配置表（ScriptableObject）
│   │
│   ├─ Bullet/            
│   ├─ Input/             ← 输入系统
│   ├─ UI/                
│   │   ├─ Scripts/
│   │   ├─ Prefabs/       ← 所有 UI 页面、弹窗
│   │   ├─ Fonts/
│   │   └─ Atlases/       ← UI 图集
│   ├─ Audio/             ← 全局音效管理 + 所有音效文件
│   ├─ Gameplay/          ← 关卡逻辑、触发器、互动物体（原来的 Triggers 可以放这）
│   └─ Rendering/         ← 后处理、自定义 Shader、Lighting 设置
│
├─ Art/                   ← 纯美术资源（不要在这里面找东西）
│   ├─ Materials/         ← 所有材质（可以再细分 PBR/URP/UI 等）
│   ├─ Textures/          ← 原始贴图（原来的 Photos/Picture 全部扔进来）
│   ├─ Sprites/
│   │   ├─ Characters/
│   │   ├─ Environment/
│   │   │   ├─ Ground/
│   │   │   ├─ Wall/
│   │   │   └─ TileSets/
│   │   └─ UI/
│   ├─ Models/            ← FBX、OBJ
│   └─ Animations/        ← 动画控制器、Animator Override（不属于特定模块的）
│
├─ Scenes/
│   ├─ Bootstrap.unity   ← 启动场景（只加载各种 Manager）
│   ├─ MainMenu.unity
│   └─ Levels/
│       ├─ Level_01.unity
│       └─ Level_02.unity
│
├─ Prefabs/               ← 全局共享预制体（不属于任何模块的）
│   └─ Shared/
│
└─ Plugins/               
│
├─ Packages/                  ← UPM 包
├─ TextMesh Pro/ 
│   ├─ Examples & Extras/
│   ├─ Fonts/
│   ├─ Shaders/
│   └─ ...（保持原结构不变）
│
└─ External/                  
```

**结构三** 不按照相关性分类，直接按照文件类型分类
